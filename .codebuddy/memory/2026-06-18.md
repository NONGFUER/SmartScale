# 2026-06-18 工作日志

## SystemInfoService 重写
- **原因**: 原实现过于复杂（JSON自持久化、手动记录boot/shutdown事件、析构写入），与系统已有的 `power-monitor.service` 功能重复
- **方案**: 改为纯只读服务，构造函数中一次性解析 `/var/log/power_monitor.log`
- **解析规则**: `[BOOT]`行数=开机次数; 连续两个BOOT间无SHUTDOWN=异常关机次数; 最后SHUTDOWN=正常关机时间
- **结果**: 代码量减少约44%（h+cpp 从 ~200行降至 ~120行），移除 QJsonDocument 依赖、recordShutdown槽、m_events列表、loadFromFile/saveToFile方法
- **涉及文件**: `src/services/SystemInfoService.h`, `.cpp`, `src/ui/components/SystemInfoDialog.qml`
