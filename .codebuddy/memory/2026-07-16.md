## checkDuplicate 判重逻辑修正：只比前一条（索引0）
**问题**：旧逻辑用 `continue` 跳过不同食材后继续往后找同食材记录——前一条是不同食材时会跨过它去找更早的同食材记录，相当于全局查找。用户反馈"没有跟前一条对比，还是全局对比"。

**修复**（`src/services/WeightHistoryService.cpp` `checkDuplicate`）：改为只取 `m_historyEntries.first()`（索引 0，最新一条记录）比较：
- 前一条是不同食材 → 直接返回不判重
- 前一条是同食材 + 重量在容差内 → 判重
- 前一条是同食材 + 重量不同 → 不判重

不再遍历整条列表。lint 零错误。
