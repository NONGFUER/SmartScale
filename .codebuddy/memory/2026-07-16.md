### NumberPadPopup Popup anchors 修复（19:36 运行时报错）
- 报错：`qrc:/.../NumberPadPopup.qml:45 Cannot assign to non-existent property "horizontalCenter"` → WorkstationPage 整页加载失败。
- 根因：`Popup` 是 QObject 派生类，**不是 Item 派生类**，没有 `anchors` 属性。原代码 `anchors.bottom`/`anchors.horizontalCenter` 非法。
- 修复：改用 `y: parent ? parent.height - height : 0` 坐标贴底（width 已等于 parent.width，水平天然占满）；同步修正 enter/exit transition 的 `to` 从错误的 `0` 改为 `parent.height - height`（贴底可见位）。
- 教训：**Popup 定位只能用 x/y，不能用 anchors**。同模块 QML 文件引用 Theme singleton 无需显式 `import SmartScale`（同 `qt_add_qml_module` 内自动可见）。lint 零错误。

## 登录错误提示不友好修复（双层防护）
用户截图反馈：登录失败时 LoginDialog 内联错误文本直接显示技术错误"网络连接失败：Error transferring https://api.shxgs.cn:5196/api/Auth/login - server replied: Bad Gateway"。**脱敏策略漏点**——`window.alert()` 有智能脱敏，但 LoginDialog 内联 `errorText` 走 `BackendAuth.loginFailed(errorMsg)` 直传 C++ 原始错误。

**根因**：`AuthService.cpp:386` 网络错误分支 `Q_EMIT loginFailed(QStringLiteral("网络连接失败：%1").arg(reply->errorString()))` 直接把 `reply->errorString()` 技术信息塞进 UI 消息——`window.alert()` 的脱敏兜不到这条路径（LoginDialog/LoginPage 是设计保留的内联错误，不走 alert）。

**修复（双层防护）**：

1. **C++ 层**（`src/services/AuthService.cpp` L373-389）：网络错误 emit 改为 `Q_EMIT loginFailed("网络连接失败，请检查网络后重试")`；`reply->errorString()` 技术细节已通过上面 `qWarning` 记入日志，可观测性不丢。
2. **QML 层兜底**：`LoginDialog.qml` / `LoginPage.qml` 各新增 `sanitizeLoginError(msg)` 函数（与 `window.alert()` 同规则：URL 替换 `<接口地址>` + 10 个技术错误特征检测 + 命中则返回"网络连接失败，请稍后重试"），`onLoginFailed` 先过脱敏再赋给 `errorText.text`。**业务友好错误**（"用户不存在"/"密码错误"/"账号已被禁用"等不含技术特征）原样保留。

**未动**：`Main.qml` 的 `__autoLoginFailedConnection`（L371-377）只打开 LoginDialog 不显示消息，错误显示由 LoginDialog 接管（已加脱敏）。

效果：用户登录失败看到的就是"网络连接失败，请检查网络后重试"，URL/SSL/HTTP 等技术细节不再外露。已写入 MEMORY.md 作为强制约束（C++ emit 错误必须友好，QML 内联错误需做脱敏兜底）。lint 零错误。
