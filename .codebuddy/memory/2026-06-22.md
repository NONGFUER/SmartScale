## 图片上传接口修复 — CustId/RecoId 参数位置错误（form-data body vs URL query）

- **症状**：图片上传接口后端收到 CustId=0, RecoId=0
- **根因**：Swagger 定义 CustId($int64) + RecoId($int64) + File($binary) **均为 multipart/form-data Request Body 字段**，但客户端代码将 CustId/RecoId 放在了 **URL query string** (`?CustId=X&RecoId=Y`) 中。后端从 body 读不到这两个字段 → 默认 0
- **修复**（`src/services/WeightHistoryService.cpp` `updateRecordImage` 约 523 行）：
  1. 移除 `QUrlQuery` / `url.setQuery()` 相关代码
  2. 新增两个 `QHttpPart` 文本字段：`name="CustId"` + `name="RecoId"`，追加到 `multiPart`
  3. File 字段保持不变（已在 form-body 中）
  4. 更新注释说明参数位置
- **附带**：移除了之前临时添加的 10s QTimer 延迟和 `#include <QTimer>`（已定位根因不再需要）
- **教训**：Swagger "No parameters" + Request body 有字段 → 参数在 body 里不在 URL query；对比 Swagger 是最快定位方式
