## 键盘明亮风格修复（暗黑→白底黑字，根因解决）
- 现象：昨天已设 `QT_VIRTUALKEYBOARD_STYLE=light` 且自定义 KeyboardStyle.qml 已部署，但重编后键盘仍暗黑。日志确认 `Cannot find style "light" - fallback: "default"`。
- **根因1（文件名）**：Qt6.8 样式查找的是 `Styles/<名>/style.qml`（入口必须叫 style.qml），不是 KeyboardStyle.qml（从 libQt6VirtualKeyboardSettings.so 字符串 `/style.qml` + Qt6.8.2 源码证实）。
- **根因2（qmldir）**：部署目录里手写的 qmldir 声明 `singleton KeyboardStyle` 但文件无 `pragma Singleton`，直接删除（样式直按文件 URL 加载，不需要 qmldir）。
- **附带修复**：原样式用了 `control.mode`（Shift/Symbol 键无此属性，仅 ModeKey 有）→ 绑定 undefined 警告。Shift 激活态改用 `control.uppercased`（BaseKey 有），symbolKeyPanel 去掉激活态高亮。
- 改动：`src/ui/vkbdstyle/KeyboardStyle.qml` → `src/ui/vkbdstyle/light/style.qml`（修正后）；app.qrc 加 alias 嵌入 `:/qt-project.org/imports/QtQuick/VirtualKeyboard/Styles/light/style.qml`（样式随二进制分发）；系统目录 `/usr/lib/.../Styles/light/` 同步替换为 style.qml（当前二进制免重编译即生效）；main.cpp 注释更新。
- 验证：重启 app 即可看到白底键盘（系统目录已修）；`make -j1` 重编后 qrc 内嵌副本兜底。

## 键盘 style.qml 加载失败 → 键盘完全消失（二次修复）
- 现象：改名 style.qml 后样式被找到，但键盘整体消失。日志：`style.qml:237: Cannot assign to non-existent property "highlight"`，随后 `keyboard.style` 为 null（`Cannot read property 'keyboardBackground' of null` 等一连串 TypeError）。
- **根因1**：`SelectionListItem` 在 Qt6.8 只是带 MouseArea 的裸 Item（无 background/text/highlight 属性），候选词 `Text` 直接放里面，`display`/`wordCompletionLength` 是上下文属性；当前项高亮用 `ListView.isCurrentItem` State。我之前照 Qt5 写法套用了不存在的属性 → 编译失败 style=null → 键盘不渲染。
- **根因2（更隐蔽）**：`KeyboardStyle` 是 QtObject，`keyboardDesignWidth/Height` **默认 0**，不显式设置则 `scaleHint=keyboardHeight/0=NaN`，所有 `N*scaleHint` 尺寸全毁。必须显式设置（用 Qt 内置同款 2560×800）。同样默认 0 需显式设置的还有 `selectionListHeight`、`alternateKeysListItemWidth/Height`。
- 修复：重写 `src/ui/vkbdstyle/light/style.qml`，严格对照 Qt6.8.2 内置 default 样式模式（github qtvirtualkeyboard v6.8.2 `src/styles/builtin/default/style.qml`），并同步部署到系统目录。
- **经验**：自定义键盘样式改完必须看日志确认无 style.qml 报错；样式加载失败不是回退而是 `keyboard.style=null` 键盘整个消失。
