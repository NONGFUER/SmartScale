# 2026-07-19

## LoginDialog 布局修复
- 问题：Dialog 加高到 640 后，按钮与输入框间距过远、样式松散。
- 根因：`StackLayout` 被固定为 `320` 高，但"账号登录"页实际内容仅约 220 高，底部约 100px 空白堆在"记住登录"与按钮之间。
- 修复：`StackLayout` 固定高度 `320 → 230`（空白缩至约 10px）；`Dialog` 总高 `640 → 560`。快捷登录页 ListView 用 `fillHeight` 填充，多历史滚动不受影响。
- 文件：`src/ui/components/LoginDialog.qml`

## 登录弹窗键盘避让修复 + 快捷登录可直接登录
- 键盘避让 bug：`Main.qml` 中 `LoginDialog.y` 误用 `inputPanel.height`（InputPanel 原始高度，未乘 `scale:0.5`），避让计算把弹窗上移过多、几乎贴顶。修复：改用 `keyboardContainer.height`（缩放后实际视觉高度）并加 `Math.max(20,…)` 兜底，与 `WifiPasswordDialog` 对齐。
- 快捷登录无法直接登录根因：`hasRememberedPassword` 仅依赖 `m_lastUserCode/m_lastPassword`（记住登录只存单账号密码），历史中其他账号无密码 → 走回退输密码分支。
- 修复：把密码随历史记录保存（`addLoginHistory` 新增 password 参数，base64 存入 `login_history.json`；`hasRememberedPassword`/`loginByHistory` 优先用历史密码，回退单账号密码）。现任意历史账号选中点登录即可直接登录。
- 文件：`src/ui/Main.qml`、`src/services/AuthService.h`、`src/services/AuthService.cpp`
