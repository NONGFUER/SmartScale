## NetworkManagerService WiFi 连接重名修复

**问题**：原先 `connectWifi()` 每次调用都无条件执行 `nmcli connection add` 创建新连接配置，导致同一 SSID 重复创建多个同名连接。

**修复内容**：
1. **头文件** (`NetworkManagerService.h`)：
   - 新增 `findExistingConnection(ssid)` 方法声明 — 查找指定 SSID 的现有 nmcli 连接
   - 新增 `deleteConnection(connName)` 方法声明 — 删除指定的连接配置
   - 新增 `m_pendingPassword` 成员变量 — 缓存密码用于失败时自动重建

2. **实现文件** (`NetworkManagerService.cpp`)：
   - 实现 `findExistingConnection()`：通过 `nmcli -t -f NAME,TYPE,802-11-wireless.ssid connection show` 查找已有连接
   - 实现 `deleteConnection()`：通过 `nmcli connection delete` 删除指定连接
   - **修改 `connectWifi()`**：在 `connection add` 前先调用 `findExistingConnection()` 检查，已存在则直接 `connection up` 复用，不存在才创建新连接
   - **修改 `onWifiConnectFinished()`**：当复用旧连接导致认证失败（Secrets were required / no valid connections）时，自动调用 `deleteConnection()` 删除旧配置，然后用缓存的密码递归调用 `connectWifi()` 重建连接

**流程图**：
```
connectWifi(ssid, password)
    │
    ├─ findExistingConnection(ssid)
    │     │
    │     ├─ 返回非空 → 已存在 → connection up 尝试激活
    │     │     │
    │     │     ├─ 成功 → ✅ 完成
    │     │     └─ 失败(密码错误) → deleteConnection() → 用缓存密码重新 connectWifi()
    │     │
    │     └─ 返回空串 → 不存在 → connection add + connection up（原有逻辑）
```

**影响文件**：
- `src/services/NetworkManagerService.h` (+6 行)
- `src/services/NetworkManagerService.cpp` (+80 行)

## 记住登录功能实现 (10:24)

**需求**：实现"记住最近登录"功能，用户勾选后下次启动自动使用保存的凭据登录，避免重复输入账号密码。

**修改文件**：

### 1. AuthService.h - 新增属性和方法
- **Q_PROPERTY**:
  - `rememberLogin` (bool) - 是否记住登录
  - `lastUserCode` (QString) - 保存的用户名
  - `hasSavedLogin` (bool) - 是否有有效的保存登录信息
- **Q_INVOKABLE 方法**:
  - `autoLogin()` - 使用保存的凭据自动登录
  - `clearSavedLogin()` - 清除记住的登录信息
- **私有方法**:
  - `loadLastLogin()` / `saveLastLogin()` / `clearSavedLoginData()`
- **成员变量**: `m_rememberLogin`, `m_lastUserCode`, `m_lastPassword`

### 2. AuthService.cpp - 持久化逻辑
- **存储位置**: `~/.config/SmartScale/last_login.conf` (QSettings INI 格式)
- **密码处理**: Base64 编码存储（简单保护）
- **构造时**: 调用 `loadLastLogin()` 加载保存的信息
- **登录成功时**: 如果 `m_rememberLogin=true` 则调用 `saveLastLogin()` 保存凭据
- **退出登录时**: 调用 `clearSavedLoginData()` 清除凭据
- **自动登录流程**: 解码 Base64 密码 → 调用 `login()`

### 3. LoginDialog.qml - UI 改动
- 新增"记住登录"复选框（密码框下方、错误提示上方）
- 复选框样式：蓝色选中态 + 自定义 indicator
- `onOpened`: 自动填充保存的用户名 + 勾选状态
- 登录按钮逻辑：如果 `hasSavedLogin` 且用户名匹配 → 直接调用 `autoLogin()`

### 4. Main.qml - 启动逻辑
- `Component.onCompleted`: 检查 `BackendAuth.hasSavedLogin`
  - 有保存信息 → 调用 `BackendAuth.autoLogin()` 自动登录
  - 无保存信息 → 打开登录弹窗
- 新增 Connections 监听自动登录失败事件，失败时回退打开登录弹窗

**用户操作流程**：
```
首次启动 → 弹出登录弹窗 → 输入账号密码 → 勾选"记住登录" → 点击登录
                                                    ↓
                                              凭据保存到本地
                                                    ↓
再次启动 → 检测到保存信息 → 自动登录(无弹窗) → 进入工作台
                                                    ↓
              (如果自动登录失败) → 打开登录弹窗让用户手动输入
```

**安全考虑**:
- 密码使用 Base64 简单编码（非明文存储）
- 前端不显示保存的密码
- 退出登录时自动清除保存的凭据
