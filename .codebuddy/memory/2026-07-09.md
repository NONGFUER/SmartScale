## AI 识别错误处理优化 + 图片去重
**PState.h**: UNKNOWN 显示文案从"识别失败 请重试"改为"--"（不再显示负面文字）

**CameraController.h/.cpp**:
- 新增 `Q_PROPERTY bool aiOnlyMode` — QML 设置后 `_processCommon` 跳过水印绘制和落盘，仅裁剪图给AI用（解决识别时产生无用水印图片问题）
- 新增 `Q_PROPERTY QString lastAiError` — 存储最近一次AI识别的错误详情，供QML弹alert用
- 网络错误：存入 `"网络请求失败：xxx（HTTP N）"`
- 业务错误(成功=false)：存入服务端 message 或默认"未能识别出食材"
- JSON解析失败："服务器响应格式异常"
- 未登录："未登录，无法进行AI识别"

**WorkstationPage.qml**:
- 识别按钮点击前设 `CameraController.aiOnlyMode = true`
- 识别完成/超时后恢复 `aiOnlyMode = false`
- `onAiRecognitionCompleted`: 失败时从 toast 改为 `window.alert()`，区分网络错误(error红) vs 业务警告(warning黄)
- 识别超时也从 toast 升级为 alert

## 修复名称卡片显示字面 "unknown" 的 bug
**根因**: `FoodTranslator::translate()` 字典未命中时返回 `PState::UNKNOWN`（即字符串 `"unknown"`），而 QML 中 `isValid()` 对有效 emsCd 返回 true → 走 translate 分支 → 显示蓝色 "unknown" 且无任何提示
**修复**: `src/utils/FoodTranslator.cpp:38` — `return PState::UNKNOWN` 改为 `return englishName`；`WorkstationPage.qml` 名称卡片 Text 组件改为 JS 块逻辑，translate 未命中（返回值===原值）时显示灰色 `"--"` 替代原始值
**影响范围**: 路径 B 的 3 个场景(API返回新emsCd无本地翻译、食材缓存损坏、人工选择异常en值)统一显示灰色 "--"，与 UNKNOWN 状态表现一致
