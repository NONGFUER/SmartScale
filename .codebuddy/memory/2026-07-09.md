## QtMqtt 从源码编译 + 集成完成
**问题**: 系统 apt 源无 Qt6::Mqtt 包，CMake 找不到 `Qt6MqttConfig.cmake`
**解决**: 从 code.qt.io 克隆 qtmqtt v6.8.2 源码 → cmake 编译 → `sudo cmake --install` 到 `/usr/local`
- 安装产物: `/usr/local/lib/aarch64-linux-gnu/libQt6Mqtt.so.6.8.2` + `/usr/local/lib/.../cmake/Qt6Mqtt/`
- CMakeLists.txt 添加: `set(Qt6Mqtt_DIR "/usr/local/lib/aarch64-linux-gnu/cmake/Qt6Mqtt" CACHE PATH "" FORCE)`

**Qt 6.8.2 API 适配（多处差异）**:
1. **`emit` 关键字禁用**: 项目定义 `-DQT_NO_KEYWORDS`，全部替换为 `Q_EMIT` (26处)
2. **SSL 方式变更**: 无 `setSslConfiguration()`/`sslConfiguration()`，改用成员变量 `m_sslConfig` 缓存 + `connectToHostEncrypted(m_sslConfig)`
3. **messageReceived 信号参数顺序**: `(const QByteArray &message, const QMqttTopicName &topic)` — 注意是 payload 在前！
4. **枚举值名称**: `SocketRemoteClosedError`→ 不存在, `BadUserNameOrPassword` 在 GCC 14 下有解析 bug（用数值 `case 4:` 绕过）
5. **方法名**: `setHost()` → `setHostname()`, subscribe/unsubscribe 参数类型 `QMqttTopicFilter`
6. **缺少 include**: 需显式 `#include <QSslKey>`; 不能在 `#include <QMqttClient>` 前加 `#include <QObject>`（触发 GCC 14 解析异常）
- 创建 `src/ui/components/AlertDialog.qml`: 支持 error/warning/success/info 四种类型、540x360 尺寸、详情展开/收起功能(Flickable)、双模式(alert单按钮 / confirm双按钮)、dangerMode 红色危险按钮
- 改造 `LogoutConfirmDialog.qml` 为基于 AlertDialog 的薄封装，保持原有 signal/method API 不变
- 在 Main.qml 挂载 globalAlert (z:9998)，暴露 `window.alert()` 和 `window.confirm()` 全局函数供子页面调用
- CMakeLists.txt 添加 AlertDialog.qml 资源

## 修复名称卡片显示字面 "unknown" 的 bug
**根因**: `FoodTranslator::translate()` 字典未命中时返回 `PState::UNKNOWN`（即字符串 `"unknown"`），而 QML 中 `isValid()` 对有效 emsCd 返回 true → 走 translate 分支 → 显示蓝色 "unknown" 且无任何提示
**修复**: `src/utils/FoodTranslator.cpp:38` — `return PState::UNKNOWN` 改为 `return englishName`；`WorkstationPage.qml` 名称卡片 Text 组件改为 JS 块逻辑，translate 未命中（返回值===原值）时显示灰色 `"--"` 替代原始值
****影响范围**: 路径 B 的 3 个场景(API返回新emsCd无本地翻译、食材缓存损坏、人工选择异常en值)统一显示灰色 "--"，与 UNKNOWN 状态表现一致

## 新增 MqttClientService MQTT 客户端工具类
- 创建 `src/services/MqttClientService.h` + `.cpp`: 基于 Qt6::Mqtt 的完整 MQTT 客户端服务类
- **核心功能**: 连接/断开/订阅/取消订阅/publish 发布消息（含文本便捷方法）
- **SSL/TLS 支持**: CA 证书加载(PEM文件)、双向认证(客户端证书+私钥)、对等验证模式(PeerVerifyMode)、ALPN 协议配置 — 完全匹配截图中的 MQTT Broker 配置界面(mqtts://user.shxgs.cn:8888)
- **自动重连机制**: 可配置间隔(ms)、最大重连次数(0=无限)，Reconnecting 状态枚举，reconnecting(attemptNumber) 信号通知
- **心跳保活**: keepAliveSeconds 配置 + 心跳监控定时器（QtMqtt 底层 PINGREQ/PINGRESP + 应用层可选心跳主题发布）
- **离线消息队列**: 未连接时 publish 自动入队，连接成功后 flushPendingMessages() 按序发送，可配置队列容量上限
- **订阅缓存**: 未连接时 subscribe 请求缓存，连接成功后自动 flushPendingSubscriptions()
- **Last-Will 遗嘱消息**: 异常断开时由 broker 代发
- **线程安全**: QMutex 保护所有共享状态，connectToBroker/disconnectFromBroker 通过 QMetaObject::invokeMethod 支持跨线程调用(QueuedConnection)
- **回调+信号双通道**: std::function 回调(onMessageReceived/onConnected/onDisconnected/onStateChanged) + Qt Signals 双重通知机制
- **QML 绑定**: Q_PROPERTY 暴露 connectionState/lastError/host/port/clientId/username/sslEnabled；Q_INVOKABLE 暴露 connectToBroker/disconnectFromBroker/subscribe/unsubscribe/publish/publishString
- **CMakeLists.txt 变更**: find_package 添加 Mqtt COMPONENTS, qt_add_qml_module 添加源文件, target_link_libraries 添加 Qt6::Mqtt

## MqttClientService 接入 shxgs MQTT Broker (user.shxgs.cn:8888)
**新增便捷接口** (MqttClientService.h/cpp):
- `initAndConnect(sn, custId)`: 一键初始化 shxgs Broker 连接（mqtts://8888），自动处理密码生成/持久化
  - 首次连接: `generateRandomPassword(16)` 生成密码，存 QSettings (`MqttShxgs`/`Password`)
  - 后续: 从 QSettings 读取复用
  - 认证: username=sn, password=持久化密码, clientid=sn
  - SSL: VerifyPeer 模式 (CA signed certificate)
  - Last-Will: `cust/{custId}/device/{sn}/up/info` → `status=offline`
- `publishDeviceInfo(sn, custId, hardver, softver, sim)`: 发布到 `cust/{custId}/device/{sn}/up/info`
  - payload 格式: `hardver=xxx&softver=xxx&sim=xxx`
- `buildDeviceTopic()` / `buildInfoPayload()`: 静态工具方法

**main.cpp 集成**:
- 创建 `MqttClientService` 实例（第84行）
- SN 就绪信号 → 自动调用 `initAndConnect(sn, authService->custId())`（第101-111行）
- 登录成功(custId变更) → 调用 `publishDeviceInfo()` 更新设备信息（第113-125行）
- QML 注册为 `"App.Backend"` 1.0 "MqttClient" 单例（第155行）
