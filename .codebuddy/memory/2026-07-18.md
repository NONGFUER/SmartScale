## 语音播报"土豆"语调不自然修复（piper 参数调优）
- **问题**：用户反馈播报"土豆"语调不标准、奇怪。
- **诊断**（espeak-ng -x -v cmn "土豆"）：音素输出 `th'u21 t'ou51` = tǔ(3声) dòu(4声)，**声调完全正确**。采样率 config 22050 = aplay 22050 也匹配。排除字音错误/变调根因。
- **根因**：piper config 的 inference 参数偏高——`noise_scale=0.667`（F0 韵律波动）+ `noise_w=0.8`（音素时长抖动），VITS 对短词（2字）的韵律随机性放大，听起来语调怪异不自然。
- **修复**（`src/hardware/VoiceSpeaker.cpp` speak 的 shellCommand）：加 piper CLI 参数覆盖（参数用**下划线**非连字符，经 deepwiki CLI 文档确认）：
  - `--noise_scale 0.333`（0.667→0.333，降 F0 波动，语调更平稳）
  - `--noise_w 0.4`（0.8→0.4，降时长抖动，节奏更稳）
  - `--sentence_silence 0.3`（0.2→0.3，句末多停顿，短词韵律更完整）
- **效果**：全局改善所有食材名播报（不只"土豆"），语调更平稳标准。warmup 命令未改（仅预热缓存，参数无影响）。
- **piper CLI 参数坑**：rhasspy/piper 1.0.0 C++ 版用下划线 `--noise_scale`/`--noise_w`/`--length_scale`/`--sentence_silence`，**不是**连字符。
- lint 通过，0 错误。用户 `make -j1` 编译后试听验证。

## 虚拟键盘暗黑→明亮（default→retro）

- **调研**：从 `/usr/lib/.../Styles/Builtin/libqtvkbbuiltinstylesplugin.so` strings 提取颜色，retro 主色 `#c5a96f`（金黄装饰）+`#ffffff`（白按键），属浅色系。retro 是 Qt 官方完整实现，零集成风险；纯白现代风需自定义 KeyboardStyle.qml（60+ 属性，集成到 qrc `/qt-project.org/imports/QtQuick/VirtualKeyboard/Styles/<name>/`，风险高），故先用 retro。
- **改动**：
  1. `app/main.cpp:97`：`QT_VIRTUALKEYBOARD_STYLE` `default`→`retro`。
  2. `src/ui/Main.qml:210`：`keyboardContainer` 容器背景 `#1E1E2E`→`#E8E8E8`（与 retro 浅色键盘一致，遮住 InputPanel scale 0.82 缩放露出的边缘）。
- 中英切换按钮 langToggle（EN 态 `#E2E8F0`/中文态 `#4361EE`）在 retro 浅色键盘上协调，无需改。

## 编译 OOM 修复（qt_add_resources→qt_add_big_resources）
- **报错**：`cc1plus: fatal error: 已杀死 signal terminated`，编译 `rcc/qrc_app_assets.cpp.o` 失败。
- **根因**：rcc 把所有图片内联成**单个** `qrc_app_assets.cpp` 巨型 C++ 文件。`workstation_bg.png`(1MB)+`lock.png`(893K)+`shuiyin.png`(64K) 等大图导致该文件巨大，编译内存峰值超主板限制被 OOM killer 杀。back2.png(4.3K) 只是触发，非主因。
- **修复**：`CMakeLists.txt:131` `qt_add_resources`→`qt_add_big_resources`（Qt6 专用大资源处理）。生成多个小 .cpp 分片编译，单文件内存峰值低不再 OOM。参数与 `qt_add_resources` 完全一致，QML 的 `qrc:/` 访问不变。
- **重新编译**：需先 `cmake ..` 重新 configure（让 CMake 重生成资源规则）再 `make -j1`。
- **可选优化**：`lock.png`(893K，一个锁图标异常大)和 `workstation_bg.png`(1MB) 可压缩减小二进制体积。

### 修正：qt_add_big_resources 用法（同日补）
- 首次改动（`qt_add_resources`→`qt_add_big_resources` 保留 PREFIX/FILES）报错 `configure_file input / is a directory`。
- 根因：`qt6_add_big_resources`（`Qt6CoreMacros.cmake:474`）是**旧式 API，签名不同**——第一参数是输出变量（非 target），`multiValueArgs` 只有 `OPTIONS`，**不支持 `PREFIX`/`FILES` 关键字**，只接受 `.qrc` 文件。误传 PREFIX/FILES 全被当 .qrc 路径解析，`/` 是目录 → configure_file 报错。
- **正确做法**：新建 `app.qrc`（项目根，列所有资源，file 路径 `resources/...` 保持 `qrc:/resources/...` 访问不变）→ `qt_add_big_resources(RCC_SOURCES app.qrc)` + `target_sources(appSmartScale PRIVATE ${RCC_SOURCES})`。
- **app.qrc 内容**：仅含原 `qt_add_resources` FILES 列表的 26 个文件。未在原列表且无 `qrc:/` 引用的文件（`lock.png`/`shuiyin.png`/`history.png`/`history_icon.png`/`camera.png`/`opr.png`）**不加**（lock.png 893K 加进去徒增体积，且 qrc 加载不必要）。
- 改资源组织方式后须清理 build（`rm -rf build/*` 或删 CMakeCache + 旧 qrc_app_assets.cpp）重新 `cmake ..` + `make -j1`。
