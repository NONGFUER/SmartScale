## 语音播报"土豆"语调不自然修复（piper 参数调优）
- **问题**：用户反馈播报"土豆"语调不标准、奇怪。
- **诊断**（espeak-ng -x -v cmn "土豆"）：音素输出 `th'u21 t'ou51` = tǔ(3声) dòu(4声)，**声调完全正确**。采样率 config 22050 = aplay 22050 也匹配。排除字音错误/变调根因。
- **根因**：piper config 的 inference 参数偏高——`noise_scale=0.667`（F0 韵律波动）+ `noise_w=0.8`（音素时长抖动），VITS 对短词（2字）的韵律随机性放大，听起来语调怪异不自然。
- **修复**（`src/hardware/VoiceSpeaker.cpp` speak 的 shellCommand）：加 piper CLI 参数覆盖（参数用**下划线**非连字符，经 deepwiki CLI 文档确认）：
  - `--noise_scale 0.333`（0.667→0.333，降 F0 波动，语调更平稳）
  - `--noise_w 0.4`（0.8→0.4，降时长抖动，节奏更稳）
  - `--sentence_silence 0.3`（0.2→0.3，句末多停顿，短词韵律更完整）
- **效果**：全局改善所有食材名播报（不只"土豆"），语调更平稳标准。warmup 命令未改（仅预热缓存，参数无影响）。
- **piper CLI 参数坑**：rhasspy/piper 1.0.0 C++ 版用下划线 `--noise_scale`/`--noise_w`/`--length_scale`/`--sentence_silence`，**不是**连字符。
- lint 通过，0 错误。用户 `make -j1` 编译后试听验证。
