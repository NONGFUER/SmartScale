## CalendarPopup 选年空白修复（StackLayout bug）

- **现象**：点"2026年7月"→点顶部"2026年 ▾"后，界面空白不显示年份网格。日志确认切换逻辑正常（pickerMode=1, decadeStart=2020），但年份视图没有 visible=true。
- **根因**：`StackLayout` 的非首项子视图（年份网格）在该 Qt 版本/布局条件下拿不到尺寸、不渲染。StackLayout 子项用裸 Item 包 anchors.fill Grid 时，非激活项高度塌缩为 0。
- **修复**：弃用 StackLayout，改用普通 Item 容器 + 两个 Grid（月份/年份）用 `visible: pickerMode===0/1` 互斥显隐，Grid 显式 `anchors.fill: parent`。
- **规则**：弹窗内两个视图互斥切换，勿用 StackLayout（非首项可能空白），用固定容器 + visible 绑定显隐更可靠。
- read_lints 0 错误，待实机验证（验证后清理诊断日志）。

## CalendarPopup ▲▼翻页按钮点不中修复
- **现象**：年月面板上翻/下翻按钮（▲▼）点击无反应。
- **根因**：与之前年份标题同类——按钮在 RowLayout 里用普通 `width:36;height:28`（非 Layout.preferredWidth/Height），RowLayout 中尺寸塌缩、MouseArea 命中区为 0。
- **修复**：两按钮改 `Layout.preferredWidth:36/Layout.preferredHeight:44`（高 28→44 更触达友好），分隔 Item 也改 Layout 形式；裸引用 pickerMode/pickerYear/decadeStart 全改 `yearMonthPicker.` 前缀。
- **规则（强化）**：RowLayout/ColumnLayout 里所有交互子项（按钮/可点文字/分隔符）的尺寸一律用 `Layout.preferredWidth/Height`，禁用普通 width/height（会塌缩导致点不中）。本弹窗年份标题、▲▼按钮均踩过此坑。
