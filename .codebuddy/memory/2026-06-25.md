  3. 语言选择器列表实际由 `VirtualKeyboardSettings.activeLocales`（可写）控制，之前从未设置。
- **验证 API**：读 `/usr/lib/aarch64-linux-gnu/qt6/qml/QtQuick/VirtualKeyboard/Settings/plugins.qmltypes` 确认 `VirtualKeyboardSettings` 单例真实属性：`locale`(rw)、`availableLocales`(只读)、`activeLocales`(rw)、`styleName`、`layoutPath`、`fullScreenMode`、`inputMethodHints` 等。**没有 `languageFilterFunc`**（我曾编造此 API，已删除）。

## 第二轮修复：activeLocales 设了但语言列表为空

- **症状**：用户编译后反馈"点地球图标，列表啥也没有"。
- **根因**：Qt 6.8.2 的 `activeLocales` 写入后内部 `updateAvailableLocales` 会重新过滤；若拼音引擎注册时机晚于 `Component.onCompleted`，`zh_CN` 被过滤掉，列表变空。
- **修复方案（已完成）**：
  1. 用 `Qt.callLater()` 延迟到事件循环空闲再设 `locale` + `activeLocales`。
  2. **关键兜底**：直接隐藏内置地球按钮。可见性由 `VirtualKeyboardSettings.visibleFunctionKeys & QtVirtualKeyboard.KeyboardFunctionKeys.Language` 控制（见 `Components/ChangeLanguageKey.qml` 源码）。
  3. 设 `visibleFunctionKeys = QtVirtualKeyboard.KeyboardFunctionKeys.Hide`（只保留隐藏键，去掉 Language）。
  4. 语言切换完全交给自定义的右上角"中/EN"按钮 + `toggleInputMode()`，调用 `VirtualKeyboardSettings.locale = "zh_CN"/"en_GB"`。
- **API 枚举值确认**（读 qmltypes）：`KeyboardFunctionKeys` 是 flag 枚举，值为 `None=0`/`Hide=1`/`Language=2`/`All=3`。**不是** `HideInputPanelKey`/`SpaceKey`（我一度写错，已纠正）。
- **待用户编译验证**：地球图标应消失，右上角"中/EN"按钮切换工作。

## 第三轮诊断：根因是系统层面缺拼音引擎（非代码问题）

- **症状**：用户反馈"还是英文"——切到 zh_CN 后键盘布局可能变了，但没有拼音候选词，输入字母直接出字母。
- **真正根因**：Debian 13 (trixie) 的 Qt6 VirtualKeyboard **不打包 Pinyin 插件**。
  - `Plugins/` 目录只有 Hangul/Hunspell/Thai 三个输入法插件，没有 Pinyin 子目录。
  - `libQt6VirtualKeyboard.so.6.8.2` strings 里的 "Pinyin" 只是枚举字符串常量，不是真的引擎类（`nm -D` 无 pinyin 符号）。
  - 这是 Debian 打包策略问题（专利/词典依赖），Qt 上游本身有 Pinyin 插件。
- **已排查的替代方案均不可行**：
  - `maliit-keyboard` 依赖 Qt5，不兼容 Qt6
  - `fcitx5` 无 virtualkeyboard 包
  - `fcitx4` 只支持物理键盘
  - apt 仓库无 `qml6-module-qtquick-virtualkeyboard-pinyin` 之类单独包
- **代码已撤销无意义改动**：
  - 撤销 `VirtualKeyboardSettings.activeLocales` 赋值（没拼音引擎时设了反而让语言列表为空）
  - 撤销 `visibleFunctionKeys = Hide`（保留地球图标，用户能看到可用语言）
  - 保留 `VirtualKeyboardSettings.locale = "zh_CN"` 和 `toggleInputMode()`，但仅作为中英键盘布局切换（无候选词）
  - main.cpp 注释明确标注"Debian 13 不打包 Pinyin，触摸屏键盘无法输入中文"
- **可行方案（需用户决策）**：
  1. 从 Qt 上游源码自编译 Pinyin 插件部署到主板（工作量中，需配 deb-src 或从 GitHub 拉）
  2. 自研 QML 拼音候选词面板 + `libpinyin15-dev` 后端（工作量大，完全可控）
  3. 接受现状：触摸屏键盘仅用于英数字，中文场景（品类搜索）改用下拉选择/语音/扫码
  - 系统已装 `libpinyin15-dev`、`libsunpinyin-dev` 可用作自研后端
- **教训**：之前我假设 "strings 出现 Pinyin = 引擎可用" 是错的，应该用 `nm -D` 验证符号或查 Plugins/ 目录才靠谱。

## 后续优化：限制地球图标只显示中英

- **用户需求**：地球图标点开只显示中文和英文，不要一堆其他语言。
