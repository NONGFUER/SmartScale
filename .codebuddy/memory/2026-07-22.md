- CellularDialog.qml 修复 4G 开关点击无反馈/状态刷新慢：原开关视觉绑定实际硬件状态 isSimOn()，底层 disableCellular 命令(mmcli 关 modem)需十几秒才回调刷新，期间视觉不变导致用户以为没生效。改为乐观更新——新增 uiSimOn/switching 本地属性，点击立即切视觉+防重复点击，并立即 refreshCellularStatus()；底层无成功信号，故监听 NetworkManager.cellularStatus 变化(onCellularStatusChanged)同步收尾，失败(onCellularOperationFailed)回滚+toast。删除未用的 isOperating()。

- 4G 状态查询新增 ip a 快速路径：新增 refreshCellularStatusFast()/findCellularInterfaceFast()/scheduleFastCellularRefresh()，命令发出后 500/1500/2500ms 用 ip a 读接口 UP/DOWN 快速刷新，避免等 mmcli 查询十几秒；只在 CellSearching 时快速更新，不覆盖 mmcli 的 Registered/Roaming 等细粒度状态。
- 修复快速路径误判：(1) 原 isUp 用 out.contains("UP") 会误匹配 LOWER_UP 载波标志，关 4G 后仍报已连接；改为以「接口有 IPv4 inet 地址 且 管理态 state UP」为数据激活判定，对 ip link down / nmcli disconnect 两种模式通用。(2) 增加 m_fastExpectEnable 方向标志：开启中无 IP 保持 Searching 不误判关闭，关闭中无 IP 立即置 Disabled；onCellularOpFinished 命令一返回先调 refreshCellularStatusFast()。(3) CellularDialog.qml 的 onCellularStatusChanged 在 CellSearching 过渡态保留乐观开关 uiSimOn，避免开关闪回。
- 用户反馈仍"像没反应"，改为纯乐观 UI（假的立即关闭）：CellularDialog.qml 点击即把 uiSimOn 与运营商文本切到目标态（operatorText 在 !uiSimOn 时立即显示"未启用"），后台 enable/disableCellular 照常异步跑；switching 仅用 fakeSwitchTimer 挡 800ms 防连点，不再卡等真实命令；onCellularStatusChanged 仅在 !switching 时才用真实状态同步（防闪回）；失败 onCellularOperationFailed 回滚+toast；onOpened 以 isSimOn() 重新同步。
