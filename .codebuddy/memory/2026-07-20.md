## 4G 弹窗重写 + 控制改 ip link set + 重启记忆

**用户需求**：CellularDialog 仿 WifiPasswordDialog 样式，24px 字体，只留运营商+启用SIM卡开关；控制命令改 `ip link set eth1 down/up`；重启记忆开关状态。

**关键前提（已实测）**：`ip link set` 需 CAP_NET_ADMIN，普通用户 sjwu 无权限（RTNETLINK Operation not permitted）。方案 = sudoers 免密 + `sudo -n ip link set <dev> up/down`。用户已确认自己执行 sudoers 配置：
```bash
echo 'sjwu ALL=(ALL) NOPASSWD: /usr/sbin/ip link set eth1 up, /usr/sbin/ip link set eth1 down' | sudo tee /etc/sudoers.d/smartscale-4g && sudo visudo -cf /etc/sudoers.d/smartscale-4g
```

**改动**：
1. `AppSettingsService` 新增 `cellularEnabled` Q_PROPERTY（QSettings 持久化，**默认 true**），key=`cellularEnabled`，文件 ~/.config/SmartScale/AppSettings.conf。
2. `NetworkManagerService`：新增 `m_sudoPath`/`m_ipPath`（构造时从候选路径解析）；`enableCellularViaDevice()`/`disableCellularViaDevice()` 整体改为 `sudo -n ip link set <dev> up/down`（删掉了原来的 nmcli connection up/device connect/disconnect 三段逻辑）；`onCellularOpFinished` 补 sudo stderr 读取 + "password is required" 专门提示"请配置 sudoers 免密"。
3. `main.cpp`：启动时若 `appSettings->cellularEnabled()==false`，延迟 3 秒（等首轮 refresh 发现硬件）自动调 `disableCellular()` 恢复上次关闭状态。
4. `CellularDialog.qml` 整体重写：modal:false+外部遮罩 z:40；顶部 back2.png+"返回"胶囊(116×44 r22)；标题"4G 移动数据"绝对居中；内容区只有两行——"运营商"(值取 NetworkManager.cellularOperator，按状态显示未启用/搜索中/未检测到模块) + "启用SIM卡"(自定义 Switch 64×36，isSimOn 按 cellularStatus 判定)；底部"关闭"按钮 180×60 r15 浅蓝底；全文字号统一 24px；删掉了原来的状态圆点/信号条/IP/Toast/hintMessage。开关点击先写 `AppSettings.cellularEnabled` 再调 enable/disableCellular，失败用 `window.toast` 提示。

**注意**：ModemManager 模式（enableCellularViaModem/disableCellularViaModem）保留未动，仍走 mmcli；只有以太网接口模式（eth1，用户实际场景）改成了 ip link set。read_lints 全部 0 错误。

## 设备信息弹窗开关合并为单灰卡 + ToggleSwitch bug 修复
- 用户要求：四个开关（价格输入/启用4G网络/启用WiFi/网络自动切换）合并进**一个**灰色圆角卡片（#F5F7FA），不用逐条分开；价格输入放最上面。
- 实现：单 Rectangle 卡片内 ColumnLayout 放 4 个 RowLayout（价格输入→4G→WiFi→自动切换），行间用 #E2E8F0 细线分隔；`SwitchRow`(自带灰背景) 改为无背景的 `ToggleSwitch`（纯 Switch + 自定义 indicator）。
- **bug 修复（关键）**：ToggleSwitch 不能在 Switch 子类上再声明 `signal toggled(bool)`（与 AbstractButton 继承的 `toggled(bool)` 重名冲突），否则组件内 `onToggled: sw.toggled(checked)` 绑定到自定义信号、用户点击永不触发，导致实例 `onToggled`（写 AppSettings）不执行——表现就是"价格输入关闭后页面没变化"，且 4G/WiFi 开关点击也不真正生效。改为直接复用 Switch 自带 `toggled(bool)` 信号（仅用户点击触发，程序赋值不触发），实例直接用 `onToggled` 写设置。
- read_lints 0 错误。
