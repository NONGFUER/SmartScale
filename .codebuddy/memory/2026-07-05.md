# 2026-07-05 工作记录

## 修复 custId/devId 雪花 ID 溢出问题

**问题**：`custId`（雪花 ID `"66885159131025408"`，17位）在 AuthService 解析时用 `toInt()` 导致溢出为 0，称重记录上传时写入错误值。

**修复范围**（4 个文件）：

### AuthService.h
- `Q_PROPERTY(int custId ...)` → `Q_PROPERTY(qint64 custId ...)`
- `Q_PROPERTY(int devId ...)` → `Q_PROPERTY(qint64 devId ...)`
- `int custId() const` → `qint64 custId() const`
- `int devId() const` → `qint64 devId() const`
- `int m_custId = 0` → `qint64 m_custId = 0`
- `int m_devId = 0` → `qint64 m_devId = 0`

### AuthService.cpp
- `custVal.toString().toInt()` → `custVal.toString().toLongLong()`
- `custVal.toInt(0)` → `custVal.toVariant().toLongLong()`
- `devId.toInt(0)` → `devId.toVariant().toLongLong()`

### WeightHistoryService.h
- `revokeRecord(int recordId, int custId, ...)` → `qint64 custId`
- `updateRecordImage(int custId, ...)` → `qint64 custId`

### WeightHistoryService.cpp
- 函数签名同步改为 `qint64 custId`
- `custId = c.isString() ? c.toString().toInt() : c.toInt()` → `.toLongLong()`
- `int uid = ...toInt()` → `qint64 uid = ...toLongLong()`
