## WeightSensorWorker SN 异常数据处理

板子未烧录 SN 时寄存器全为 `0xFF`，解析成 Latin1 是 `'ÿ'` 乱码。

在 `src/hardware/WeightSensorWorker.cpp` 的 `modbusReadSN` 函数解析段（约 448-476 行）追加两层校验：

1. 全 `0xFF` 检测 → 返回空串，打印告警 `[Modbus] SN 未烧录(全 0xFF)，返回空串`
2. 解析后含不可打印字符（`!ch.isPrint()`）→ 返回空串，打印非法字节

返回值仍为 `0`（通信成功），调用方应通过 `sn->isEmpty()` 判断是否未烧录。建议 UI 层做空串占位显示（如 "----"）。

## LogoutConfirmDialog 圆角修复

`src/ui/components/LogoutConfirmDialog.qml` 的 `Dialog` 直接写 `radius: 16` 会报错（`Dialog`/`Popup`/`Item` 都没有 `radius`，该属性是 `Rectangle` 独有）。修复方式：删除注释掉的 `//radius: 16`，改为给 Dialog 添加 `background: Rectangle { radius: 16; color: "#FFFFFF"; border.color: "#E2E8F0"; border.width: 1 }`。

同时此文件「退出登录」按钮原本只有 `Layout.fillWidth: true` 无下限，会被压扁导致文字裁切；已补 `Layout.minimumWidth: 100` 保证最小宽度。

## QML 约定记忆

- `Dialog`/`Popup`/`Item` 等非 `Rectangle` 类型无 `radius` 属性，需要圆角时必须通过 `background: Rectangle { radius: N }` 实现。
- `RowLayout`/`ColumnLayout` 中使用 `Layout.fillWidth: true` 时，建议同时设置 `Layout.minimumWidth`，防止内容被压缩到无法显示。

## RecordDetailDialog 支持左右滑动/箭头切换上下记录

在 `src/ui/components/RecordDetailDialog.qml` 详情弹窗的图片区域实现记录导航：

- 新增对外接口：`property var recordList: []`（父组件传入完整记录列表）+ `signal navigateToRecord(int newIndex)`
- 新增只读计算属性：`currentIndex`（通过 `recordTime` 匹配 recordList 定位，-1 表示未匹配）、`hasPrev`、`hasNext`
- 图片区域加入：左/右半透明圆形箭头按钮（‹/›，无 prev/next 时置灰禁用）、覆盖全区的滑动手势 `MouseArea`（右滑>80px→上一条，左滑<-80px→下一条，垂直滑动占主导时忽略）、右上角计数角标（`当前/总数`）、底部"左右滑动切换记录"提示条
- 箭头/滑动均通过 `navigateToRecord(currentIndex ± 1)` 触发，由父组件切换 `currentDetailRecord`，详情内容随绑定自动刷新
- `WorkstationPage.qml` 中 `RecordDetailDialog` 调用改为：`recordList: WeightHistoryService ? WeightHistoryService.historyEntries : []` + `onNavigateToRecord` 回调更新 `root.currentDetailRecord`
- 所有导航元素仅在 `recordList.length > 1` 时显示，单条/空记录时不干扰原交互
