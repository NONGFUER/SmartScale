### 技术要点备忘
- **nmcli device show 合法字段**：GENERAL, CAPABILITIES, INTERFACE-FLAGS, WIFI-PROPERTIES, AP, WIRED-PROPERTIES, WIMAX-PROPERTIES, NSP, IP4, DHCP4, IP6, DHCP6 等。**不含 WIFI.SIGNAL / CONNECTIONS.WIFI-SSID**
- **nmcli -t 输出格式**：所有字段都是 `field.name:value` 格式，必须用冒号分割取后半部分
- **GENERAL.CONNECTION ≠ 真实 SSID**：前者是连接 profile 名（可能被 sanitize 截断），需用 `connection show <name>` 反查
- **Qt 资源路径**：CMake PREFIX="/" 下文件仍保留子目录路径，实际映射为 `qrc:/` + CMake FILES 中的相对路径（含 resources/icon/ 前缀）。需以 `qrc:/resources/icon/x.png` 访问，不是 `qrc:/x.png`

18. **4G 双模式支持（ModemManager + 以太网接口）** — 适配 eth1 类型的 4G USB 模块：
    - **背景**: 用户设备的 4G 模块以以太网模式工作（接口名 `eth1`），无 ModemManager 服务，原 mmcli-only 方案完全无法检测
    - **NetworkManagerService.h 新增**:
      - `Q_PROPERTY bool hasCellularHardware` — QML 可绑定属性，表示是否检测到 4G 硬件
      - `discoverCellularDevice()` — 发现 4G 网络接口设备（策略: GSM连接配置名匹配→候选接口列表eth1/usb0/ppp0→enx* USB设备）
      - `enableCellularViaDevice()` / `disableCellularViaDevice()` — nmcli 接口模式下的启用/禁用
      - `updateCellularStatusFromNmcli()` — 从 nmcli device show 输出解析 4G 状态
      - `updateHasCellularHardware()` — 更新硬件检测状态并发射信号
      - 成员变量: `m_cellularDeviceName`(缓存接口名), `m_hasCellularHardware`
      - 信号: `cellularHardwareChanged(bool)`
    - **NetworkManagerService.cpp 核心改造**:
      - `refreshCellularStatus()`: 双模式检测 — 优先 ModemManager(mmcli) → fallback 网络接口(nmcli device show) → 都失败标记 Disabled+无硬件
      - `enableCellular()`: 双模式 — 有 modem 走原有逻辑 → 无 modem 但有网络接口走 enableCellularViaDevice()
      - `disableCellular()`: 同上双模式
      - 接口模式信号强度默认给 65%（无法从 eth 接口获取真实值）
    - **CellularDialog.qml hasModem() 重写**: 从复杂的 Disabled+错误信息推断改为直接读取 `NetworkManager.hasCellularHardware`
    - **关键发现**: nmcli 设备列表显示 `eth1:ethernet:connected`，说明 4G 模块被识别为普通以太网接口
