6. **修复 WiFi 列表中已连接网络不显示状态的问题（问题3）**
   - **根因**：`WifiListDialog.onOpened` 直接调用 `scanWifiNetworks()`，但此时 `NetworkManager.wifiSsid` / `wifiStatus` 还是过时值（或空），导致 delegate 中 `isCurrentNetwork` 比较 `modelData.ssid === NetworkManager.wifiSsid` 永远为 false
   - **修复**：`WifiListDialog.qml:430-438` — onOpened 改为先调用 `refreshWifiStatus()` 确保当前连接状态数据就绪，再用 `Qt.callLater()` 延迟触发扫描，保证状态在列表渲染前已同步

7. **修复 nmcli device show 返回空输出（WIFI.SIGNAL 字段不合法）**
   - 根因：`nmcli -f WIFI.SIGNAL device show` 不是合法字段（合法字段列表不含此字段），导致 exitCode=2 整个命令失败
   - 修复：改用合法字段 `GENERAL.STATE,GENERAL.CONNECTION,IP4.ADDRESS`

8. **修复 SSID 名称显示为连接 profile 名而非真实 SSID**
   - 根因：`GENERAL.CONNECTION` 返回的是 nmcli 连接配置名（经 sanitizeConnectionName 去掉了中文/特殊字符），如 "吴赛杰的iPhone" → "iPhone"
   - 修复：三步流程 — 发现设备 → 查状态获取 connName → 用 `nmcli connection show <connName> -f 802-11-wireless.ssid` 反查真实 SSID

9. **修复 SSID 带 "802-11-wireless.ssid:" 前缀的问题**
   - 根因1：`-t` 模式输出格式是 `key:value`，需取冒号后部分 → 新增 `extractSsidValue()` 静态方法
   - 根因2：`connection show` 可能返回多行重复值 → 只取第一行

### 技术要点备忘
- **nmcli device show 合法字段**：GENERAL, CAPABILITIES, INTERFACE-FLAGS, WIFI-PROPERTIES, AP, WIRED-PROPERTIES, WIMAX-PROPERTIES, NSP, IP4, DHCP4, IP6, DHCP6 等。**不含 WIFI.SIGNAL / CONNECTIONS.WIFI-SSID**
- **nmcli -t 输出格式**：所有字段都是 `field.name:value` 格式，必须用冒号分割取后半部分
- **GENERAL.CONNECTION ≠ 真实 SSID**：前者是连接 profile 名（可能被 sanitize 截断），需用 `connection show <name>` 反查
- **Qt 资源路径**：CMake PREFIX="/" 下文件映射为 `qrc:/` + 文件名部分（不含 resources/icon/ 前缀）
