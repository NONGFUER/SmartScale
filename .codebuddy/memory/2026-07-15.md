## 图片圆角 + 加高修复
- **问题1（无圆角）**：`Rectangle.clip:true` 只裁矩形包围盒，**不跟随 radius** → Image 四角仍是直角盖住灰瓦圆角。Qt6.8 实测确认。
- **修复**：改用 `MultiEffect` mask 方案（Qt6 标准）：
  - 源 `Image{visible:false}` + 遮罩 `Rectangle{id:imgMask; radius:12; color:"#FFFFFF"; visible:false; layer.enabled:true}` + 显示项 `MultiEffect{source:foodImg; maskEnabled:true; maskSource:imgMask; anchors.fill:foodImg}`
  - 关键：source 与 maskSource 都 `visible:false`，mask 须 `layer.enabled:true` 才能渲染成纹理；mask 用 alpha 通道，白色圆角矩形=可见区
  - 删除 imageContainer 的 `clip:true`（mask 已处理圆角，clip 反而在 hover 缩放时切出直角）
  - hover `scale` 从 Image 移到 imageContainer（整瓦缩放，圆角随之缩放不露直角）
- **问题2（太矮）**：`cellHeight 160→200`，图片高 `cellHeight-70` 由 90→130(+44%)，卡片总高 192 仍 < cellHeight 200 留 8px 行距

## 二次加高 + 加大卡片间距
- `cellHeight 200→250`；图片高 `cellHeight-70→cellHeight-80` = 170（+40，且增长慢于 cellHeight 以腾出间距）
- 卡片宽 `cellWidth-12→cellWidth-18`（水平间距 12→18）
- 间距公式：垂直 gap = cellHeight - (image + spacing12 + name~30 + margins20) = cellHeight - image - 62 = 250-170-62 = 18
- 结果：图片 170px、水平/垂直间距均 18px

## CategoryCorrectionDialog 默认选中第一级第二级首个分类
- **症状**：点击「选择食材」打开弹窗后，左侧一级分类默认有高亮（selectedTopIndex=0），但中间二级品类列表始终空白（selectedSubCateId=""），右侧网格也不显示食材。
- **根因**：`onOpened` 把 `selectedTopIndex` 设为 0 后才异步调用 `fetchIngrCategories()`；数据到达后触发 `onCategoryTreeChanged`，但该函数首行 `if (dialogRoot.selectedTopIndex >= 0 || CategoryService.categoryTree.length === 0) return`——因 `selectedTopIndex=0` 使 `>=0` 恒为 true，整段「默认选中第一个一级+第一个二级」逻辑永不执行。
- **修复**：将判断条件改为「二级品类已选才跳过」（`if (dialogRoot.selectedSubCateId !== "") return`）。这样：弹窗刚打开（二级空）→ 自动选中第一级 + 第二级首个；用户手动点击某二级后 → 不再覆盖。
- 文件：`src/ui/pages/CategoryCorrectionDialog.qml`（`Connections onCategoryTreeChanged`，约 829 行）。

## AI 推荐候选复用图片卡片（删除独立 ListView）
- **背景**：之前推荐模式（selectedTopIndex=-999）用一个独立的黄色 ListView delegate（序号圆圈+中文名+code）展示候选，与正常分类模式的图片卡片风格不一致。
- **需求**：候选的展示直接复用现有的图片卡片（GridView + FoodCard delegate）。
- **改动**（`src/ui/pages/CategoryCorrectionDialog.qml`）：
  1. `getDisplayItems()` 新增推荐模式分支：遍历 `recommendCandidates`（格式 `{code,name}`），用 `UserIngredientService.findByEmsCd(code)` 反查完整食材 item（含 `img`/`imgLocal`/`id`/`cn`/`en`），反查到则直接 push 完整 map（卡片显示真实图片）；反查不到则 push 适配后的 `{cn:name, en:code, id:"", img:"", imgLocal:""}`（卡片显示占位文字）。
  2. 删除独立的 AI 推荐模式 ListView（原约 60 行，含黄色卡片/序号徽章/recMouse 等）。
  3. GridView 去掉 `visible: selectedTopIndex !== -999 || ...` 条件，改为始终可见——推荐/分类/搜索三种模式统一复用 GridView + FoodCard delegate，`getDisplayItems()` 按模式返回不同数据。
- **边界**：反查不到的候选选中后 `selectedIngrId=""`，确认按钮（`enabled: selectedLabel !== "" && selectedIngrId !== ""`）不可用，避免提交无效 id。
- **字段映射**：候选 `{code,name}` → FoodCard `{en:code, cn:name, id, img, imgLocal}`；`code` 即 AI 返回的 emsCd（参见 WorkstationPage.qml:1235 `findByEmsCd(newPred)` 用法）。
