## 修复识别与保存链路核心 bug + aiDetected 标志位改造

### 1. AI 响应 JSON 解析 bug（CameraController.cpp:805-813）
- **根因**：在线 AI 接口返回 `{"data":{"code":"tudou","name":"土豆"},"success":true}`，`data` 是嵌套对象，原代码 `obj["data"].toString()` 对 QJsonObject 调用返回空串，触发 `isEmpty()` 误判为识别失败 → label 被置为 `unknown`。
- **修复**：先判断 `dataVal.isObject()`，新格式取 `dataObj["code"]` 作为 emsCd，旧格式（data 直接为字符串）兼容保留。同时提取 `message` 字段用于错误诊断日志。

### 2. 保存按钮缺品类/ingrId 校验（WorkstationPage.qml 保存按钮）
- **缺陷**：原 `enabled` 被注释掉（恒为 true），识别失败且未手动选择时直接保存 → `ingrId=""` → `buildUploadJson` 的 `toLongLong("")` 失败 → 上传 `ingrId=0` 的脏数据。
- **修复**：`onClicked` 内增加三道硬校验：①重量 ≤0.01 拦截；②`PState.isValid(currentPrediction)` 品类无效拦截；③`currentIngrId` 为空拦截。每道都有 toast 提示。

### 3. AI 回调识别失败时残留脏 currentIngrId（WorkstationPage.qml onAiRecognitionCompleted）
- **缺陷**：识别返回 unknown 时未清空 `currentIngrId`，上一次有效识别的 ingrId 残留，下次保存会误用。
- **修复**：识别无效时主动清空 `currentIngrId`；反查食材库失败时打 warn 日志并置空。

## WorkstationPage 白屏修复（ActionButton 跨目录 import 缺失）
