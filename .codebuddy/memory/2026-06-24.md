## 修复未登录状态下点击"识别"按钮的拦截缺失

- **文件**：`src/ui/pages/WorkstationPage.qml` 识别按钮 `recognizeMA.onClicked`（约 750 行）
- **缺陷**：原逻辑直接调用 `CameraController.captureVegetable()` 触发识别，未校验 `BackendAuth.currentUser`，导致未登录用户也能发起识别请求。
- **修复**：在 `onClicked` 最前面（重量校验之前）加入登录拦截范式，与同文件"保存"按钮一致：
  ```qml
  if (!BackendAuth.currentUser) {
      console.warn("[WSP] 未登录，拦截识别操作，弹出登录窗口")
      window.toast("请先登录后再识别", "warning", 2000)
      window.showLogin()
      return
  }
  ```
- **登录引导**：弹 warning Toast + 调用 `window.showLogin()` 打开登录弹窗（与保存按钮拦截范式完全一致）。
- **参考范式**：保存按钮 `onClicked`（约 917 行）已有完整的登录拦截写法，可作为后续新增敏感操作的模板。
