## 修复识别与保存链路核心 bug + aiDetected 标志位改造

### 1. AI 响应 JSON 解析 bug（CameraController.cpp:805-813）
- **根因**：在线 AI 接口返回 `{"data":{"code":"tudou","name":"土豆"},"success":true}`，`data` 是嵌套对象，原代码 `obj["data"].toString()` 对 QJsonObject 调用返回空串，触发 `isEmpty()` 误判为识别失败 → label 被置为 `unknown`。
- **修复**：先判断 `dataVal.isObject()`，新格式取 `dataObj["code"]` 作为 emsCd，旧格式（data 直接为字符串）兼容保留。同时提取 `message` 字段用于错误诊断日志。

### 2. 保存按钮缺品类/ingrId 校验（WorkstationPage.qml 保存按钮）
- **缺陷**：原 `enabled` 被注释掉（恒为 true），识别失败且未手动选择时直接保存 → `ingrId=""` → `buildUploadJson` 的 `toLongLong("")` 失败 → 上传 `ingrId=0` 的脏数据。
- **修复**：`onClicked` 内增加三道硬校验：①重量 ≤0.01 拦截；②`PState.isValid(currentPrediction)` 品类无效拦截；③`currentIngrId` 为空拦截。每道都有 toast 提示。

### 3. AI 回调识别失败时残留脏 currentIngrId（WorkstationPage.qml onAiRecognitionCompleted）
- **缺陷**：识别返回 unknown 时未清空 `currentIngrId`，上一次有效识别的 ingrId 残留，下次保存会误用。
- **修复**：识别无效时主动清空 `currentIngrId`；反查食材库失败时打 warn 日志并置空。

### 4. aiDetected 标志位改造（贯穿 7 个文件）
- **背景**：用户指出 `json["aiDet"] = !record.categoryName.isEmpty()` 错误——AI 识别和手动选择 categoryName 都有值，导致 aiDet 永远 true。必须按"是否通过 AI 识别接口得出"区分。
- **改动文件**：
  1. `src/data/models/WeightRecord.h`：新增 `bool aiDetected = false`
  2. `src/data/models/WeightRecord.cpp`：`toMap`/`fromMap` 增加 `aiDetected` 字段
  3. `src/data/repositories/WeightRecordRepo.cpp`：`insert`/`update` SQL 增加 `ai_detected` 列 + bindValue；`fromQuery` 读取 `ai_detected`
  4. `src/data/DatabaseManager.cpp`：建表 DDL 增加 `ai_detected INTEGER DEFAULT 0`；新增迁移 v3 `ALTER TABLE weight_records ADD COLUMN ai_detected INTEGER DEFAULT 0`
  5. `src/services/WeightHistoryService.h`：`addRecord` 末尾新增 `bool aiDetected = false` 参数
  6. `src/services/WeightHistoryService.cpp`：`addRecord` 赋值 `model.aiDetected`；`buildUploadJson` 改为 `json["aiDet"] = record.aiDetected`（替换原 `!record.categoryName.isEmpty()`）；ingrId 无效时增加 qWarning
  7. `src/ui/pages/WorkstationPage.qml`：新增 `currentAiDetected`/`pendingSaveAiDetected` 属性；AI 回调成功反查时设 `currentAiDetected=true`，失败时置 false；`onLabelConfirmed`（人工选择/纠错）设 `currentAiDetected=false`；保存按钮暂存 `pendingSaveAiDetected`；`addRecord` 调用传入第 7 参数；`onPhotoSaved` 清理时重置两个属性
- **判定规则**：仅当品类由 `CameraController.onAiRecognitionCompleted` 回调直接得出 → `aiDetected=true`；任何人工选择/纠错路径 → `aiDetected=false`。

### 5. PState::label() 状态提示拆分（src/core/PState.h:32-37）
- **缺陷**：原 `label()` 把 IDLE/NOT_READY/UNKNOWN/NONE 四种语义不同的状态统一返回"点击选择食材"，会误导用户。
- **修复**：按状态分别返回：IDLE→"请识别或选择食材"、NOT_READY→"设备未就绪"、UNKNOWN→"识别失败 请重试"、NONE→"无识别结果"。文本控制在 4-7 字以适配 88px 大字渲染。
