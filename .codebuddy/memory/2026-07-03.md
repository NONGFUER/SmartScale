## Token 无感刷新机制完整实现

### 问题诊断
- **P0 并发竞态**：`refreshToken()` 无锁，多 Service 同时触发会发出多个刷新请求
- **P0 无 401 拦截**：全部 5 个使用 Token 的网络回调均不区分 HTTP 状态码
- **P1 预检覆盖不全**：仅 WeightHistoryService 有 `isTokenExpiringSoon()`
- **P1 无统一协调器**：各 Service 各自管理（或完全忽略）刷新状态

### 架构要点
- 双层防竞态：Service 本地 `m_refreshing` + AuthService 全局 `m_isRefreshing`
- 统一信号 `tokenRefreshCompleted(bool,QString)` 驱动所有排队请求自动重发
- 连续失败阈值 `kMaxRefreshFailures=2`，超过建议重新登录
