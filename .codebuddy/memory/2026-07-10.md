# 2026-07-10

## MQTT 设备信息上报修复（app/main.cpp）
- 问题：设备信息(sn/custId)上报只在 `AuthService::userInfoChanged` 触发，且要求 sn 与 custId 同时就绪；但两者异步独立到达（sn 来自 WeightSensor 串口读取的 `snChanged`，custId 来自登录响应）。若登录早于 SN 读回，则 `sn.isEmpty()` 为真直接 return，SN 到达后 `initAndConnect` 也不调 `publishDeviceInfo`，导致设备信息永远空、后端收不到。
- 修复：抽出 `tryPublishDeviceInfo` lambda，同时挂在 `snChanged` 与 `userInfoChanged` 上，两者都齐了才上报，并加诊断日志（打印缺哪个）。
- 注意：admin 离线登录 custId 恒为 0，按设计不上报云端。遗留：initAndConnect 用当时 custId 设遗嘱主题，SN 早于登录时 will 为 cust/0/...，极端情况，未修。
