## AuthService refresh token 接口报 "The refreshToken field is required."
- **现象**：后端返回 `The refreshToken field is required.`（不是前端的"未登录或无刷新令牌"），说明请求已发出但后端认为 body 里 refreshToken 缺失/为空
- **改动**（`src/services/AuthService.cpp`，诊断 + 防御）：
  1. `refreshToken()` 入口：`isEmpty()` → `trimmed().isEmpty()`，拦截纯空白字符串
  2. `tryRefreshToken()` 入口：加二次防御 + 打印 `m_refreshToken.len`/`prefix` + 完整 `[HTTP] Body` 日志（与登录请求一致）
  3. `parseAuthResponse()`：解析 `refreshToken` 后若为空，打 warning 打印 `data.keys()` 并检测是否含 `refreshToken`/`RefreshToken` 字段（排查原因 A：后端字段缺失或大小写不一致）
- **待用户复跑确认根因**：
  - 原因 A（最可能）：后端登录接口未返回 `refreshToken` 字段，或字段名大小写不一致
  - 原因 B：离线登录后误触发 refresh（已被新防御拦截）
  - 原因 C：createApiRequest 带了过期 Authorization header（代码看 REFRESH_TOKEN 调用未传 token，理论无此问题）

## AuthService refresh token 接口 — 真正根因修复（2026-06-21 续）
- **根因**：从 Swagger 截图确认，后端 `[FromBody] string refreshToken` 期望 **JSON 字符串值**（带引号的纯字符串 `"token"`），而不是 JSON 对象 `{"refreshToken":"token"}`
- **修复**（`src/services/AuthService.cpp` tryRefreshToken）：把 `QJsonObject bodyObj; bodyObj["refreshToken"]=...;` 改为 `QByteArray bodyData = "\"" + m_refreshToken.toUtf8() + "\"";`
- 这解释了之前为什么报 `The refreshToken field is required.`：后端把 body 当 string 反序列化，整个对象 `{"refreshToken":"..."}` 不是合法 string → 视为空
